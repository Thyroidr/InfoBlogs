APIO 2010 Patrol

Posted on 2018-12-12 Edited on 2019-04-17 In Misc. Problems

Task Description

Problem statement can be found on the APIO Website.

To sum up the task, we are given an unweighted tree and are asked to add $k$ edges, such that the minimum distance to traverse every node at least once is minimised after adding the $k$ edges (all added edges are to be traversed exactly once).

One thing we can notice immediately is that for $30 %$ of the marks, $k=1$, and that for the remaining $70 %$, $k=2$. It appears as if the first subtask is designed to guide people to solve subtask 2, which is indeed the case.

We shall look at the $k=1$ case first. One can notice that when an edge is added to a tree, a cycle is formed. Since we have to traverse the added edge exactly once, the number of steps saved is exactly the size of the cycle saved take away one, since we have to traverse the added edge. It follows that we should find the longest path on the tree to maximise the saving. This can be done in $O(n)$ time using DP.

Introduction to Competitive Programming

Posted on 2018-11-14 Edited on 2019-04-17 In Miscellaneous

Overview

Competitive programming, known also as informatics olympiad for secondary students, is an umbrella term for a wide range of competitions. Competitions under this term examine the participants' understanding in classic computer science areas including graph theory, data structures, and dynamic programming and ability to apply the knowledge. It also encompasses areas of discrete mathematics.

From my personal experience, participating in competitive programming aids greatly with problem solving abilities and the ability to write bug-free code quickly. The knowledge required for competitive programming are also often times part of Computer Science courses at undergraduate or graduate levels.

Combinatorics Tricks

Posted on 2018-10-27 Edited on 2019-04-17 In Number Theory and Mathematics

Assumed Knowledge

  • Combinatorics Function: for $n$ balls, the number of different combinations consisting of $m$ balls can be given by:

    $$ C_n^m = \frac {n!} {m!(n-m)!} $$

  • For a range of size $n$, the number of ways to divide it into $m$ ranges (empty ranges are not allowed) is given by $C_{n-1}^{m-1}$ (in this article $C$ refers to the combinatorics function given above). The situation is equivalent to having $n-1$ gaps and picking $m-1$ of them.

Virtual Contest 2018/10/19

Posted on 2018-10-25 Edited on 2019-04-17 In DTOJ

Q1 & CodeForces 147B: Smile House

Problem Link

Solution

We can rephrase the problem to:

for a given graph of $n$ vertices, find a cycle with the minimum of vertices such that the sum of all edge weights on the cycle is positive.

Let $f_{i, j, k}$ be the maximum distance from $j$ to $k$, passing $i$ vertices. then clearly $f_{i, j, k} = \min{f_{i-1, j, l} + g_{l, k}}$ (memories from Floyd-Warshall). Preprocessing this would have a time complexity of $O(n^4)$, we can hence consider changing $f$ to a sparse table. Let $f_{i, j, k}$ represent the maximum distance from $j$ to $k$, passing $2^i$ vertices, then $f_{i, j, k} = \min {f_{i-1, j, l} + f_{i-1, l, k}}$. This would have a time complexity of $O(n^3 \log n)$.
