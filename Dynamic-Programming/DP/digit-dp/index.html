Digit DP

Posted on 2018-09-14 Edited on 2019-04-17 In Dynamic Programming

Description

Digit DP is a type of dynamic programming often associated with stunningly high data ranges (by that, I mean $10^{100000}$). It is often used for counting the number of integers satisfying specific constraints in the range $l$ to $r$. As implied by its name, digit DP is based on digits: units, tens, hundreds, etc. Digit DP, in its essence, is brute-force enumeration with memorisation.

Explaination

Enumerating like this involves no memorisation and is very slow. To speed up, we can enumerate digit by digit, starting from the higher digits. Let's say that $r = 213$, for which we will start in the hundreds place. All integers that are within our consideration need to have $0$, $1$ or $2$ in their hundreds place (whether or not we include $0$ depends on the problem, specifically, if there are any constraints on the number of digits possessed by the number).

Furthermore, when we move to the tens and units place, the number we are enumerating can never exceed $213$. What this means is, if we picked $1$ for the hundreds place, the tens place can be anything from $0$ to $9$, since the hundreds place $1$, ensures that the number can not exceed $213$. However, if we picked $2$ for the hundreds place, then the tens place can only be $0$ or $1$. Similarly, if we picked $2$ for the hundreds and $1$ for the tens, then the units place can only be $0$ to $3$.
