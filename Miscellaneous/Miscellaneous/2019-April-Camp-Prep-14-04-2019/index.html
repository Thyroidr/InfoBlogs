Persistent Segtree Practice

The general idea of persistent segtree is to never modify the nodes of a segtree, and instead of create new nodes, such that historical versions can be accessed. One can build a fenwick tree on top of a persistent segtree, which would allow for modification of "history".

Luogu P2468

Giving a $R \times C$ matrix, and $M$ queries, the $i-th$ of which has the form:

what is the least number of elements from the sub-rectangle $(x_{1, i}, y_{1, i})$ to $(x_{2, i}, y_{2, i})$ needed such that their sum is greater than equal to $h_i$?

For subtask $1$ ($50%$): $R, C \le 200, M \le 200,000$

For subtask $2$ ($50%$): $R = 1, C \le 500,000, M \le 20,000$

Additionally all elements of the matrix are positive and no greater than $1000$.

For subtask $1$ we can find the number of elements in each sub-rectangle, that is greater than $k$, and their sum in $O(RC)$, with a constant factor of $1000$. This allows us to binary search the answer. The second subtask can be solved with both persistent segtree and square root decomposition. For the sake of practice here I used persistent segtree (it is also lower in time complexity).

Luogu P3157

Given an array $A$ of $N$ elements, let an inverse pair be defined as a pair $(i, j)$ where $1 \le i < j \le N$ and $A_i > A_j$. $M$ elements will be deleted in the order given from $A$, output the number of inverse pairs in $A$ before each deletion.

$1 \le M \le N \le 100,000, M \le 500,000$

First use a fenwick tree to record for each $i$, the number of inverse pairs it forms with elements before it, and the number of inverse pairs it forms with elements after it. The number of inverse pairs we delete when we delete $i$ will not be greater than the sum of these two elements, and the different is just the number of inverse pairs it makes with already deleted elements. To find the number of inverse pairs it makes with already deleted numbers we can use a persistent segtree in a fenwick tree. Notice that for $i$ to make an inverse pair with a deleted $j$, $j$ is either smaller than $i$ and after $i$, or greater than $i$ and before $i$. Here we treat each root of a node in a fenwick tree, each prefix of the fenwick tree stores not a value, but a segtree. The segtree represented by the $k$-th prefix represents all the deleted elements with position up to $k$, and each segtree is sorted to allow us to query the number of elements whose value is in a range.

Introduction to Convex Hull Trick (CHT)

Check this out CodeForces Blog

APIO 2014 Split the Sequence (Luogu P3648)

The problem statement can be found at PDF.

Observe that no matter how the parts are divided, each final part would have always been multiplied with the rest of the sequence. We are therefore only concerned about what are the final parts. Let $s_i$ denote the prefix sum up to $i$, and $f_{i, k}$ denote the maximum score in the prefix up to $i$, with $k$ divides, we have,
$$f_{i, k} = \max_{0 \le j < i}{f_{j, k - 1} + s_j(s_i - s_j)}$$

Note that this is a linear function of $s_i$, and we may hence use CHT to optimise the DP. Since $s_i$ is non-decreasing, the CHT runs in amortised constant time. This gives a solution of $O(NK)$.
