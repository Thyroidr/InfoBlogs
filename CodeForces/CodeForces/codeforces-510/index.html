A: Benches

Problem Link
First find the most seated bench, $Max$, in the beginning. The maximum possible $k$ is $Max+m$. With $Max$ we can find how many people can be seated without increasing the maximum number seated in a bench. Fist let this many people be seated such that all benches are seated by the same number of people, and then it is pigeonhole principle.

C: Array Product

Problem Link
Obviously since we would like our product to be as large as possible, we would not want to delete any positive integers. We also only want to delete one negative integer if there is an odd number of them, this would be the smallest negative integer in absolute value. Furthermore, we want to delete all zeros. Hence, we first group all zeros together (and the negative integer, if need be), delete the resulting integer, and then multiply the remaining integers together. Be careful not to do the deletion step if all integers are scheduled for deletion - we need one integer in the end.

D: Petya and Array

Problem Link
Let $p_i= \sum_{j=1}^i a_j$, then the problem transforms to find the number of pairs $p_i$ and $p_j$ such that $i < j$ and $p_j - p_i < t$. We can assign each prefix sum a rank (their position when sorted in ascending order). For $p_r$, all prefix sums satisfying the requirement need to be $p_l$ where $l<r$ and the rank of $p_l$ needs to be higher than the rank of $p_r - t$. We can use a binary indexed tree with lower_bound to complete this query. (I initially used std::set to do this, which did not work since std::set iterators are bidirectional instead of random-access, meaning distance(it1, it2) takes $O(n)$ time).
