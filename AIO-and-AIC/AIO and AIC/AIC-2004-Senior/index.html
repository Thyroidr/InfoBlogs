Zig-Zag Cipher

Problem Link

The solution to this problem is just pure simulation. Let the length of the encrypted message be $l$, declare a matrix $M$ of size $r \times l$. We can then label the cells in $M$ that should be filled in elements of the encrypted message. We can then fill in $M$ with the encrypted message and output.

Atlantis

Problem Link

Let $src$ denote the ocean outside the map, then we can notice that whether or not a cell is covered in water in the end is based on the minimum bottleneck path from $src$ to that cell. Precisely, let $x$ be the minimum bottleneck from $src$ to a cell, the cell is covered in water iff $x < f$ or ($x < h$ and $x > Map_{cell}$). If $x < f$, then obviously the water level of $f$ would be able to cover the cell. If $x < h$ and $x > Map_{cell}$, then the cell would be covered by the water level of $h$, and that not all water can escape from the cell. The solution for this problem hence involves building the graph, finding the minimum bottleneck tree (remember that the minimum spanning tree of a graph is also a minimum bottleneck tree of the graph), and finding the bottlenecks.

Bouncy Ball

Problem Link

We may rephrase the problem to: find the lexicographical $k$-th sequence among all sequences ${ S_1, S_2, S_3, \ldots }$, satisfying that for all $i$ and valid $j$,
$$\sum_{S_{i, j} \in S_i}{S_{i, j} = n} \text{ and } S_{i, j + 1} - S_{i, j} \ge a$$

Let $f_{i, j}$ denote the number of sequences whose sum is $i$ and starting element is $j$, and has an acceleration of at least $a$. We can have,
$$f_{i, j} =
\begin{cases}
1, & \text{$i=j$} \
\sum_{l=j+a}^{i-j} f_{i-j, l}, & \text{$i \neq j$} \
\end{cases}
$$

To understand the DP equation, notice that when $i=j$, the starting element is equal to the sum of the sequence, so the sequence is made up solely of the first element. When $i \neq j$, the sequence has a length of at least $2$, and we may consider the sequences if we remove the first element $j$. If $j$ is removed, the sum becomes $i - j$, and that the starting element has to be from $j + a$ to $i - j$. We can simply find the number of sequences whose sum is $i-j$ and starting element is in the range $[j + a, i - j]$.

The question now is how is $f$ going to help in solve the problem, and the answer is, we can use $f$ for binary searching our answer. Take the sample input for example, the 10 sequences are:

  1. 1, 2, 3, 4
  2. 1, 2, 7
  3. 1, 3, 6
  4. 1, 4, 5
  5. 1, 9
  6. 2, 3, 5
  7. 2, 8
  8. 3, 7
  9. 4, 6
  10. 10

Notice that there are $5$ sequences starting with $1$, or that $f_{10, 1}=5$, and that we are looking for the $6$th sequence. The final sequence hence cannot start with $1$. We look at $2$ instead and realise that $f_{10, 1} + f_{10, 2} = 7$, the sequence hence starts with $2$.

Since we have determined that the first element of the sequence is $2$, the remaining elements need to have a sum of $10 - 2 = 8$. The remaining elements also needs to form a sequence starting with at least $2+1=3$. Since we know $f_{10, 1}=5$, we are looking for the $1$st sequence, starting with at least $3$, and summing to $8$.

  1. 1 2 5
  2. 1 3 4
  3. 1 7
  4. 2 6
  5. 3 5
  6. 8

Above are the sequences that sum to $8$, among them, only $5$ and $6$ start with $3$ or above. The remaining elements elements of our sequence are hence ${ 3, 5 }$, and that our answer is hence ${2, 3, 5}$.

The above process shows us iterating over each digit of the sequence and using the $f$ array to determine each digit. This would have a time complexity of $O(n^3)$. If we use binary search to optimise for each digit, the time complexity becomes $O(n^2 \log n)$. If we then use binary indexed tree to store the $f$ array, the time complexity becomes $O(n \log^2 n)$. Processing the $f$ array using binary indexed tree also requires a time complexity of $O(n^2 \log n)$.
