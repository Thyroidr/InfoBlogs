Union Find Set

Posted on 2018-09-21 Edited on 2019-04-17 In Data Structures

Description

Union find set is a tree-based data structure that allows two operations:

  1. query: given $x$, find the root of the tree in which $x$ belongs.
  2. merge: given $x$ and $y$, merge the trees in which they belong (if they do not belong to the same tree).
    (Usually all elements are the root of their own tree in the beginning)

This allows us to answer problems such as:

Outside your hotel window is the town plaza, on which several mafia families have gathered. There are $n$ people in total on the plaza, we know that $m$ pairs of them belong to the same family (different pairs may belong to different families, one individual may be in multiple pairs). Determine the maximum number of mafia families outside.

To maintain a union find set, we utilise an array recording the father of each element. This array is initialised such that the father of each element is itself. When we perform a query for the root of element $x$, we query for the root of the father of $x$, since the root for the two is the same, and that the father is closer to the root. Optimisations

There are two major optimisation techniques for union find set, path compression and union by rank.

Path Compression

This optimisation technique is the most often used and is more effective than union by rank. It involves a simple change to the getroot function.

As one may notice, this destroys the structure of the tree and compresses it to at max. 2 layers. This allows for a much faster query. There are situations, however, where we wish to preserve the structure of the tree.

Union by Rank

Union by rank essentially means that in addition to the father array, we would also maintain a rank array, recording, for each element, the height of its subtree. When we merge two trees, we also merge the shorter one to the taller one. This ensures that the resulting tree has at max. $log n$ layers. Union by rank perserves the tree, despite being slower than path compression.
