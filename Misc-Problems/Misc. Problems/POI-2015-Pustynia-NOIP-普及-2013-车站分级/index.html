POI 2015 Pustynia & NOIP 普及 2013 车站分级

Posted on 2018-10-07 Edited on 2019-04-17 In Misc. Problems

Problem Description

The problem "Pustynia" from the Polish POI 2015 is essentially the same problem as the problem "车站分级" from the Chinese NOIP 普及 2013. You can find the original problem in Polish and in Chinese. The Chinese version does have a smaller data size and can be solved with naive methods. Below is the English Translation of the Chinese version.

On a one way rail line, there are $n$ stops labelled from $1$ to $n$. Every stop has a level no lower than $1$. There are multiple train routes using this rail line, all of which satisfy the requirement below:

if a train stops at stop $x$, then it is required to stop at all stops with a level greater than or equall of that of $x$ between its departing and terminal stops (including the departing and terminals).

Below is a example situation:

Stop ID 1 2 3 4 5 6 7 8 9
Train \ Level 3 1 2 1 3 2 1 1 3
1 Start -> Stop - > Stop Term.
2 Start -> -> Stop Term.
3 Start -> -> -> Stop -> -> -> Term.
4 Start Stop Stop Stop Stop Term.
5 Start -> Stop -> -> -> Term.

All trains except the fifth satisfy the requirement (since stop 3 and 6 are both level 2, it cannot stop at stop 3 without stopping at stop 6).

Given $m$ trains and their respective stops, find a way to assign the $n$ stops a minimum number of levels such that all trains are legal.

Input

The first line of input contains $n$ and $m$.
In the preceding $m$ lines, the $i$-th line contain a positive integer, $s_i$, the number of stops of the $i$-th train. $s_i$ is followed by $s_i$ integers in ascending order, representing the stops of this train.

Output

A integer, the minimum number of levels that can be assigned to the stops with all trains being legal.

Solution

Before thinking about the solution, we need to first rephrase the problem: given a positive integer sequence of $n$ integers, and $m$ requirements, what is the largest integer in the sequence. We will also define a requirement as: given a number of elements in the sequence, these elements are strictly larger than all other elements in between them.

This problem is now clearly a constrain satisfaction problem. The naive method may be for each element in a constraint, connect it with all elements in the constraint region (but are not in the constraint) with an directed edge of weight $1$, and then find the shortest path from each vertex to a vertex without incoming edge. This has a clear time complextiy of $O(n^2)$.

We can notice that the constrainting effect of all elements in a constraint are "grouped", before applying to the elements in the constraint region. This means that for each constraint, we can create a new vertex, connect it with directed edges of weight $0$ from the constraint elements. We can then connect the new vertex to all constrainted elements with an edge of weight $1$. This has a time complextity of $O(mn)$.

Notice that for each constraint, the constrainted elements are consecutive. We can use a segment tree accelerate the graph construction. Each father node has a edge of weight $0$ to its children. This would accelerate the graph construction, and has a time complexity of $O(m log n)$.
