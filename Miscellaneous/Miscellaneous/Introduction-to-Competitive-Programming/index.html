Introduction to Competitive Programming

Posted on 2018-11-14 Edited on 2019-04-17 In Miscellaneous

Overview

Competitive programming, known also as informatics olympiad for secondary students, is an umbrella term for a wide range of competitions. Competitions under this term examine the participants' understanding in classic computer science areas including graph theory, data structures, and dynamic programming and ability to apply the knowledge. It also encompasses areas of discrete mathematics.

From my personal experience, participating in competitive programming aids greatly with problem solving abilities and the ability to write bug-free code quickly. The knowledge required for competitive programming are also often times part of Computer Science courses at undergraduate or graduate levels.

Competitive programming involves participants attempting to solve a number of questions in a limited time. Usually they are required to code a solution to the questions using a programming language they know. The source code of the solution program is to be submitted and compiled into executable programs by the organisers. The executable programs would then be run against a number of inputs, their outputs would be compared with the correct outputs. If the outputs match, the participant would score a number of points.

An example problem for competitive programming can be found at AIO Training Page.

C++ is arguably the most used programming language for competitive programming for its performance and STL (C++ Standard Template Library), which includes templates of basic data structures and algorithms. The popularity of C++ can be seen in the status page of CodeForces (one of the largest competitive programming websites).

Competitions

The competitive programming competitions can be categorised into informatics olympiads, ACM-ICPC, and general competitions, respectively for secondary students, tertiary students, and the public.

Informatics Olympiads

Informatics olympiads are usually organised in the unit of nations or regions, and for the purpose of selecting a national team to participate in the International Olympiad in Informatics. Olympiads are usually held on-site instead of online. Depending on the purpose and skill level of the participants, the olympiads have different difficulties. Some of the informatics olympiads include:

  • International Olympiad in Informatics
  • Asia-Pacific Informatics Olympiad
  • Australian Informatics Olympiad
  • Australian Invitational Informatics Olympiad
  • French-Australian Regional Informatics Olympiad
  • USA Computing Olympiad
  • National Olympiad in Informatics in Provinces (China)
  • National Olympiad in Informatics (China)

ACM-ICPC

ACM-ICPC is a set of on-site competitions for undergraduate students globally. More information can be found at the ACM-ICPC website

Other

Other competitions are usually held online, and have a wide range of participants. A large proportion of participants in these competitions hope to use them to prepare for informatics olympiads or the ACM-ICPC. Some of the websites organising these competitions include CodeForces, TopCoder, and CodeChef. One can also organise competitions using these websites.

Areas to Study for Competitive Programming

As mentioned, competitive programming mainly examines the participants' proficiency in data structures, discrete mathematics, dynamic programming, and graph theory. The competitions usually place a far greater emphasis in the application of these skills and problem-solving skills.

Data Structures

A data structure is an organisation allowing for the storage, modification, and access of data. Take an integer array (a list of integers accessed by their index) for example, we can change and access the value stored at a particular index, as well as adding new values to the array.

Data structures can be far more complex than this, and can achieve far more than this. Some data structures used in competitive programming include Union Find Set, Segment Tree, and Binary Indexed Tree.

Discrete Mathematics

Discrete mathematics refers the the branches of mathematics that are discrete rather than continuous. A discrete value cannot be broken down beyond a certain unit, unlike continuous values. For example, we can have 1 person, 2 people, but not 1.5 people. However, we can have 1 degree Celsius, 2 degrees Celsius, 1.5 degrees Celsius, and even 1.55555... degrees Celsius.

Via most definitions, the whole of theoretical computer science, including data structures and graph theory, is included in discrete mathematics. Here discrete mathematics refers to areas including combinatorics, probability, game theory, and number theory.

Dynamic Programming

Dynamic Programming, or DP, is an important technique in competitive programming. It involves breaking a larger problem down to sub-problems, solving the sub-problems, and collecting their solutions to form the solution to the larger problem.

DP, unlike many other techniques, is listed independently for its effectiveness and difficulty (to me). The application of DP is often highly complex and involves a thorough understanding in both the nature of the problem and other components of theoretical computer science.

Graph Theory

A graph is a mathematical structure modelling pairwise relationships. This can be used to model maps, organisational structures, and your friendship circle. Questions under graph theory include the shortest path from one point to another, the maximum amount of "water" that can flow from one place to another, and etc.

Graph theory is used in informatics to model the problem, often not literally, and to aid seeking the answer. Graph theory problems (just like all other problems) require the participant to be flexible in applying their knowledge.
