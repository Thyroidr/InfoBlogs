Conditional Statements & Operators

Posted on 2018-09-09 Edited on 2019-04-17 In Basic C++

Introduction

"I'll go to civic this afternoon if you go as well", sentences like this are common in real life. They are call Conditional Statements in computer science. We can reword the example sentence to:

`If "you are going to civic this afternoon", then "I will go to civic this afternoon", else "I will not go to civic this afternoon".`

And that is exactly how conditional statements work in C++, though of course, typing above sentence into C++ gets you a compiler error. Abstractly, the above sentence can be presented as,

If the statement within the brackets is true (returns a Boolean value of 1), then lines in the first pair of large brackets are executed, else the lines in the second pair of large brackets are executed.

Boolean Operators & Statements

In order to use if statements, we would first need to know how to write statements in C++, since "you are going to civic this afternoon" obviously is not something your computer can understand. Let's start with the operators,

`a == b means "is a equal to b?"` `a = b means "assign the value of b to a"`
It is crucial to separate the "==" operator and the "=" operator. The first one would return a Boolean value: true if a is equal to b, false otherwise. We have other operators which return Boolean values:
`a > b means "is a larger than b?"` `a >= b means "is a larger than or equal to b?"`
`a < b means "is a smaller than b?"` `a <= b means "is a smaller than or equal to b?"`
There are three extra operators that are used to express different things
`'!' changes true to false, false to true` `'&&' means "and"` `'||' means "or"`
With these operators we can write the following statements,
`a == b && c > a meaning "Is a equal to b and c larger than a?"` `! a == b || b == c meaning "Is a not equal to b and b equal to c?"` `Note : a != b is equivalent to ! a == b `
## Arithmetic Operators & Expressions ## There are 5 arithmetic operators in C++, corresponding to addition, subtraction, multiplication, division and modulus (remainder of division) in Maths.
`a = b + c means "assign a the sum of b and c"` `a = b - c means "assign a the result of b - c"` `a = b * c means "assign a the product of b and c"` `a = b / c means "assign a the result of b divided by c"` `a = b % c means "assign a the remainder of b dividing by c"` `None of the above expressions change the value of b or c, and that all of a, b and c are integer type variables.`

Practice - Problem 4A on CodeForces

To test your understanding of conditional statements, go to CodeForces, register an account, and complete problem 4A. You can also find the problem description and solution below.

4A - Watermelon

time limit per test:1 second | memory limit per test: 64 megabytes
input: standard input | output: standard output
One hot summer day Pete and his friend Billy decided to buy a watermelon. They chose the biggest and the ripest one, in their opinion. After that the watermelon was weighed, and the scales showed w kilos. They rushed home, dying of thirst, and decided to divide the berry, however they faced a hard problem.

Pete and Billy are great fans of even numbers, that's why they want to divide the watermelon in such a way that each of the two parts weighs even number of kilos, at the same time it is not obligatory that the parts are equal. The boys are extremely tired and want to start their meal as soon as possible, that's why you should help them and find out, if they can divide the watermelon in the way they want. For sure, each of them should get a part of positive weight.

Input

The first (and the only) input line contains integer number w (1 ≤ w ≤ 100) — the weight of the watermelon bought by the boys.

Output

Print YES, if the boys can divide the watermelon into two parts, each of them weighing even number of kilos; and NO in the opposite case.

Example

Input: 8 | Output: YES
