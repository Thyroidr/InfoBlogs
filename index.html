IOI 2008 Pyramid Base

Posted on 2019-04-12 Edited on 2019-04-17 In IOI

Let us consider the $B = 0$ and $B \neq 0$ cases separately. When $B \neq 0$, we may binary search the answer. Suppose we are trying to the minimum cost to fit a square of dimensions $x \times x$ on the board, we only need to find a cell that could be the bottom right corner of the square, we call these cells eligible . Let us consider how each existing rectangle determines which cells are ineligible. Notice that we just need to expand the rectangles by $x - 1$ downwards and rightwards, and that would be precisely the cells made ineligible by this rectangle. We may do this for all rectangles and then use a sweepline in conjunction with a segtree to find the minimum cost. The time complexity for this is $O(n \log^2 n)$.

This does not solve the $B = 0$ case in full though, as we need a $O(n \log n)$ solution. Let us now define a range $[l, r]$ on the x-axis to be eligible if and only if we can place rectangle in the range, such that the rectangle spans $[l, r]$ and the other dimension is greater than equal to $r - l + 1$. Observe that if $[l, r]$ is eligible, then $[l + 1, r]$ and $[l, r - 1]$ are also valid. We may hence use sliding window (two pointers but on one array) with a segtree to devise a $O(n \log n)$ solution.

FARIO 2019 Papa and Baby Frog

Posted on 2019-03-29 Edited on 2019-04-17 In FARIO

It is obvious that we should make two forests based on the array. For node $i$ to be a valid home, we need that there exists a node $j$, such that,

  1. $j$ is an ancestor of $i$ in the papa forest.
  2. $j$ is in the subtree of $i$ in the baby forest
  3. the distance from $i$ to $j$ is no greater than $K$ in either forests.

Notice that in a euler tour, node $j$ is in the subtree of node $i$, if and only if all appearances of $j$ have at least one appearance of $i$ both to the left and right. We may hence iterate over the papa forest, using a stack to keep track all of a node's ancestors in increasing order of depth. We keep all ancestors within a $K$ distance of our current node active in a segtree built on the euler tour, and identify if any of these nodes satisfy the condition using the segtree.

FARIO 2019 Coloured Walkway

Posted on 2019-03-29 Edited on 2019-04-17 In FARIO

First consider each colour as a node on a graph, and that whenever we reach a colour, we update the node represented by the colour. Update here is defined as finding the sum of all adjacent nodes and adding that to our node. This should allow us to get first $3$ subtasks.

Consider the fourth subtask, where we have a tree. Here we could use some DP to avoid having to collect the sum of the children nodes. In essence, for each colour $i$ we have $f_i$, which stores the number of ways to reach any tile with colour $i$. We also have $childSum_i$, which stores the number of ways to reach any of $i$'s children nodes. When we attempt the update $i$, we add $f_{\text{i's parent}}$ and $childSum_i$ to $f_i$, we then add the same value to $childSum_{\text{i's parent}}$ to keep it up to date.

Regarding the fifth subtask, we could adopt a similar approach as the fourth subtask, except that now we have a regular graph. We may turn this graph to a DAG by making all edges directed, from a node of lower degree to a node of higher degree and solve it like subtask $4$. I cannot prove the time complexity of this.

IOI 2008 Fish

Posted on 2019-03-20 Edited on 2019-04-17 In IOI

Obviously it is important to consider where a set of jewels could have ended up in. Clearly a set of jewels may end up with multiple fishes. It is hence important to remove the ambiguity, and only consider the case where a set of jewels lands in longest fish possible. Notice that the longest fish a set of jewels can end up in must be the longest fish with a certain kind of jewels. We can hence calculate the answer by considering the longest fish with each kind of jewel. Let $E(J_i, J_x)$ denote the number of fish with jewel $J_x$ that can be eaten by the longest fish with jewel $J_i$. We have three cases:

  • If the longest fish with jewel $J_i$ is longer than the longest fish with jewel $J_x$, the we may have $0$ to $E(J_i, J_x)$ jewels, in a set of jewels with $J_i$, and it would still be mapped to $J_i$.
  • If the longest fish with jewel $J_i$ is not longer than the longest fish with jewel $J_x$, but $E(J_x, J_i) < E(J_i, J_i) + 1$, then we can only have $0$ to $E(J_i, J_x)$ jewels of type $J_x$ is we have $E(J_i, J_i) + 1$ jewels of type $J_i$.
  • If the longest fish with jewel $J_i$ is not longer than the longest fish with jewel $J_x$, and $E(J_x, J_i) \ge E(J_i, J_i) + 1$, then no set with jewel $J_x$ can be mapped to the longest fish with kind $J_i$.

The above observations will allow for a $O(K^2)$ solution. If we then notice some monotonicity related to the above cases and do some sorting, we can use segtree to reduce the time complexity down to $O(F \log F)$.

IOI 2008 Islands

Posted on 2019-03-20 Edited on 2019-04-17 In IOI

The fact that there are $V$ nodes and exactly $V$ edges, and the way bridges are created, imply that the islands form a pseudoforest. The question is simply asking for the sum of the longest path in each pseudotree. Notice that we can consider a pseudotree as a cycle, with some trees attached. The longest path for a pseudotree may fall into one of the following two cases:

  1. The entire path is within a tree.
  2. The path starts in some tree, goes into the cycle, and ends in some other tree.

For the first case, we can just find the diametre of each tree. For the second case, we can find the deepest node in the tree attached to node $i$, where node $i$ is on the cycle. Let this be $s_i$, we can iterate clockwise and DP using $s_i$ to find the longest path for the second case.

IOI 2008 Teleporter

Posted on 2019-03-20 Edited on 2019-04-17 In IOI

The key observation in this problem is the model may be illustrated clearly if we consider each interval, rather than teleporter, as nodes in a graph, and consider teleporters as edges. In this case, there is always a path from the leftmost interval to the rightmost interval, and that all other connected components are simple cycles. Further observation tells that adding a teleporter can transform a cycle to a part of the path. This adds all the edges in the cycle to the path, allow with $2$ other edges. Alternatively, using $2$ additional teleporters we can add $4$ edges to the path, and using only $1$ teleporter can trivially add $1$ edge to the path.

IOI 2008 Linear Garden

Posted on 2019-03-15 Edited on 2019-04-17 In IOI

To find out the answer, we could instead find the number of sequences that came before our given sequence. Since L comes before P in the alphabets, all sequences that came before our given sequence has to share some prefix (possibly of length $0$) with our sequence. Formally, a balanced sequences $t$ comes before our sequence $s$ iff at some position $i$:

  • $s_i = P$
  • $t_i = L$
  • $\forall j \in [1, i - 1], t_j = s_j$

This allows us to find the answer by finding all $i$ such that $s_i = P$, and finding the number of sequences satisfying the above condition at position $i$. Let us consider L to be $-1$ and P to be $1$. Obviously, the sum of any suffix of any prefix of our sequences has to in $[-2, 2]$. Let $f_{i, j, k}$ denote the number of different suffix for range $[i, n]$, given the minimum suffix of $[1, i - 1]$ is $j$ and the maximum suffix is $k$. Since $-2 \le j \le k \le 2$, we have $15n$ states, and we can use DP to fill in $f$ in $O(N)$.

Beware of the memory limit though, recursive DP may result in MLE or RE depending on implementation.

IOI 2008 Type Printer

Posted on 2019-03-14 Edited on 2019-04-17 In IOI

Consider first how to minimise the number of operations if we have to delete the last word. In this case, it is clear that sorting these words lexicographically minimises the number of operations. We know that sorting strings lexicographically is equivelant to adding them to a trie and DFS on the trie. In the case of the trie, not having to delete the last word simply means that we don't need to get back to the root of trie for our DFS. Hence, we can choose to end our DFS at the longest word, which minimises number of edges (or operations) that we go through. We can translate this back to just sorting the strings.
