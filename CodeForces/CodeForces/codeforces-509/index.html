1041A: Heist

Problem Link
Since we are looking for the minimum possible number of keyboards, we can assume that the keyboard with lowest index, $Min$, is the first keyboard and the keyboard with the highest index, $Max$ is the last keyboard. This means that before the heist there would be at least $(Max-Min+1)$ keyboards in the store. $((Max-Min+1)-n)$ is hence the answer.

1041B: Buying a TV Set

Problem Link
Since the width and height of the purchased TVs have to be integers, we can divide $x$ and $y$ by $gcd(x,y)$ (greatest common divisor) to find the 'units' of width and height. We can then use $a$ and $b$ to find respectively how many 'units' are there in the size limit. Let $x'$ be the unit for width and $y'$ be the unit for height, the answer is
$$min(\frac{a}{x'}, \frac{b}{y'})$$

As of finding $gcd(x,y)$, we can use Euclidean's algorithm (essentially, $gcd(x,y) = gcd(y, x % y)$) for GCD.

1041C: Coffee Break

Problem Link
This problem is interesting in that it is not hard to solve (greedy + priority queue for my solution), but that it is hard to prove the greedy algorithm is correct. We will start from the finding the minimum number of days.

The problem seeks to decompose a set, $A$, of distinct integers into subsets where any pair of elements in a subset have a difference higher than $d$. Without a loss of generality, we can sort $A$ into ascending order.

Here we define two elements, $a_i$ and $a_j$, are in order iff $i < j$ and $a_j - a_i > d$. We will also define a chain as a subset where all elements are in order, and an antichain as a subset where no two elements are in order. We can hence transform the problem to finding the minimum number of chains in set $A$.

We can hence apply Dilworth's Theorem to $A$. Dilworth's Theorem states that

In any finite partially ordered set, the maximum number of elements in any antichain equals the minimum number of chains in any partition of the set into chains.

We can hence know that for this problem,
$$\text{minimum number of days} = \text{minimum number of chains} = \text{maximum length of antichain}$$

We can find the maximum length of antichain with DP, but this does not address the second question, which asks for the elements in each chain in our decomposition. Let us define $a_i \in A$ to be special iff there are no elements to the left of $a_i$ smaller than $a_i-d$. We can remove all special elements from $A$ and add them to another set, $X_1$ (all elements in $X_1$ are in the same order as they were in $A$). It is easy to show that $X_1$ is an antichain. We can repeat this process on $A$ to form $X_2$. All elements in $X_2$ must be in order with the first element in $X_1$, otherwise they would have either stayed in $A$ or been added to $X_1$. We can repeat this process until $A$ is empty and we have obtained $X= \{ X_1, X_2, \cdots \}$. Using $X$, we can decompose the original $A$ into chains.

We shall also notice that $X_1$ is a prefix of $A$, and that $X_2$ is a prefix of $A \backslash X_1$ (this implies that $X_1$ and $X_2$ are continuous subsets of $A$). We hence only need to enumerate through $A$ once, to both generate $X$ and decompose $A$.

1041D: Glider

Problem Link
The tutorial explains this problem really well. The key is that it is always optimal to jump in the beginning of any air segment, $s$, since moving $+1$ or $-1$ from $s$ do not increase the answer. We can hence enumerate each beginning of airflow, using $lower_bound$ to calculate the answer from each $s$.

(I used upper_bound, which makes no difference since the air segments do not intersect)
