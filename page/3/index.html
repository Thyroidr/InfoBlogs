FARIO 2018 Greed

Posted on 2019-01-14 Edited on 2019-04-17 In FARIO

Problem Link

Whenever we see minimise the maximum or maximise the minimum, binary search is almost always the way to go, as is in this problem (with the exception of graph theory, where it is about half-half between bsearch and MST). With bsearch in this problem, we would be directly searching for the answer, and our decision problem would be whether we could divide the cake such that no piece is larger than $lim$ in no more than $k$ cuts.

This could be solved with DP + two pointers. Let $f_{i, j}$ be the minimum number of cuts needed at the $i$-th horizontal cut and the $j$-th vertical cut.
$$f_{i, j} = min(f_{i, t_c}, f_{t_r, j}) + 1, t_c \le j, t_r \le i, \text{difference in area} \le lim$$

Notice the as $t$ increases, $f_{i, t}$ and $f_{t, j}$ are non-decreasing. Hence, for calculating $f_{i, j}$, we would wish to minimise $t$, while adhering to the area limit. Iterating lexicographically (by rows and then columns) allows us to use two-pointers to make the time complexity of finding $t$ amortised constant.

FARIO 2017 Optimal Delivery

Posted on 2019-01-13 Edited on 2019-04-17 In FARIO

Problem Link

Notice that if we consider the weight of each edge as an unknown, we have $4900$ unknowns. Whenever we make a query, we receive a linear equation. For every unique pair of starting and ending location, we introduce a new unknown to the system of linear equations. The number of unique pairs hence cannot exceed $100$ since we have atmost $5000$ queries available.

It may be tempting to try to use matrices to solve this system of linear equation. I have failed at that lol. Below is my solution.

Let us just consider how to solve the edge weights for a $2 \times 2$ grid. Let us denote the top edge as $a$, left edge as $b$, right edge as $c$, and bottom edge as $d$. We could query $ab$, $ac$, $bd$, and $cd$, and denote each of their results as $R_{\text{edges involved}}$. Notice that $a - d = ((ac - bd) + (ab - cd)) / 2$ and $c - b = ((ac - bd) - (ab - cd)) / 2$. Given this, we only need the sum of $a$ and $d$ or $c$ and $b$ to find their edge weights.

Doing this for all $2 \times 2$ squares require $4 \cdot 49 \cdot 49 = 9604$ queries, we hence wish to optimise this. Notice the redundancy here. Iterate over the rows, at row $x$, iterate over the columns, at column $y$, query for the path that starts $(x, 0)$, moves straight east to $(x, y)$, down to $(x + 1, y)$, then straight east to $(x + 1, C)$. This allows us to find all $ab$ and $ac$ in $2450$ queries. Do the same for $bd$ and $cd$ to use another $2450$ queries. We now have the difference in edge weight, between any two parallel edges in the same row or column.

Now, if we could find the edge weight of a horizontal or vertical edge, we could find all edge weights in the column or row, respectively. We query for a path that starts at $(0, 0)$ and goes straight east to $(0, C)$, as well as a path starts at $(0, 0)$ and goes straight down to $(R, 0)$. We then make one query for each row and column of edges to find all horizontal and vertical edge weights in that row or column, respectively. As of what queries to make, that is left as an exercise to reader cus I am too lazy to write. Alternatively, the reader could read the code.

This solution uses exactly $5000$ queries. For the subtask where $Q = 4999$, the reader either make some observations or read the code.

FARIO 2017 Pyramid Cake

Posted on 2019-01-13 Edited on 2019-04-17 In FARIO

Problem Link

Let $k_{i, j}$ denote the minimum height in the rectangle whose top-left corner is $(0, 0)$ and bottom=right corner is $(i, j)$. Notice that $k_{i, j} = \min (k_{i - 1, j}, k_{i, j - 1})$. This observation allows us to solve the problem easily using DP. Let $f_{i, j}$ denote the maximum volume that could be built in the rectangle whose top-left corner is $(0, 0)$ and bottom-right corner is $(i, j)$. We have,

$$ f_{i, j} = \max (f_{i - 1, j} + k_{i, j} j, f_{i, j - 1} + k_{i, j} i)$$

FARIO 2017 Pair Programming

Posted on 2019-01-13 Edited on 2019-04-17 In FARIO

Problem Link

Let us consider each person as an edge between two nodes, one as the editor and one as the language. This way we get a graph (not necessarily connected). Notice that on a tree with $k$ edges, there exists at least one way to pair edges that share nodes, such that we have $\lfloor \frac{k}{2} \rfloor$ pairs. We could hence just break each connected component into trees by creating new copies of nodes when adding an edge creates a cycle. This way we have obviously maximised the number of pairings in each connected component, and consequently the whole graph.

FARIO 2016 Hooliganism

Posted on 2019-01-09 Edited on 2019-04-17 In FARIO

Problem Link

We are essentially asked to assign a total ordering to the vertices, such that for any vertex u, the number of vertices with a high order than $u$ and is directly connected to $u$ is lower than or equal to $a_u$.

Let $k_x$ be the number of remaining edges for vertex x. If $a_x \le k_x$, it is obviously optimal to place $x$ with the lowest remaining ordering. If we cannot find any $x$ satisfying this, then it is impossible to achieve the task since we have to place a vertex with the lowest remaining ordering.

Time complexity: $O(V + E)$

FARIO 2015 The Nightlight

Posted on 2019-01-08 Edited on 2019-04-17 In FARIO

Problem Link

After some observation, we notice that at any cell u, we can move to a cell in its bottom right. If the destination, v, has the same colour as u, the cost is no more than the distance - 1. If v has a different colour to u, the cost is no more than the distance. We notice that doing the former saves us 1, when compared to doing the latter, and we hence wish to maximise the number of times we do the former. We also notice that if we always do the latter, our cost is simply the distance from origin to target. Our answer is hence the distance from origin to target, substracted by the maximal number of times we do the former.

At cell u, we can query for all cells in its top-left, with the same colour as u, for the maximum saving. If we iterate in lexicographical order, we reduce the query down to 1D, which we may answer with a segtree for each colour. This has a space complexity of $O(r \log r \times k)$. If we then discretise the coordinates for cells with the same colour, the space complexity reduces down to $O(rc log rc)$.

FARIO 2014 Super Maria II

Posted on 2019-01-08 Edited on 2019-04-17 In FARIO

Problem Link

Let $f_i$ denote the minimum cost to find the prince, when Maria is standing just outside a range of size $i$ and facing the range. We have,

$$f_i = \min_{1 \le j \le i} \max(f_{j-1}+c_1, f_{i-j})$$

Notice that $f_{j-1}$ is monotonically non-decreasing, and $f_{i-j}$ is monotonically non-increasing. $j$ is hence obviously the intersection point of these two functions. Notice that as $i$ increases, for each $j$, the $f_{i-j}$ does not decrease. This graphically implies that the intersection point does not move left as $i$ increases. We could hence use two-pointers to find the intersection point. This allows us to find all elements of $f$ in $O(n)$ time complexity.

While finding $f$, we could store the intersection point for each $i$. This will allow us to backtrack our answer.

FARIO 2014 Connect

Posted on 2019-01-07 Edited on 2019-04-17 In FARIO

Problem Link

Let us consider arrangements such as 1313 where two pairs overlap as edges between node 1 and 3. For any pair of nodes x, we could construct edges to all pairs (with one node in between the x's and the other to the left of the x's). We could do so by iterating from left to right, whenever we encounter the left node of a pair, construct edges to all marked nodes in between it and its right node. We then mark its right node. Since we are always constructing edges to marked nodes in range, we could use segtree to improve the time complexity to $O(n log n)$.
