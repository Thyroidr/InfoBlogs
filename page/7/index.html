Flow Network

Posted on 2018-09-17 Edited on 2019-04-17 In Graph Theory

Maximum Flow

Scenario

Problem Link
Every time it rains on Farmer John's fields, a pond forms over Bessie's favorite clover patch. This means that the clover is covered by water for awhile and takes quite a long time to regrow. Thus, Farmer John has built a set of drainage ditches so that Bessie's clover patch is never covered in water. Instead, the water is drained to a nearby stream. Being an ace engineer, Farmer John has also installed regulators at the beginning of each ditch, so he can control at what rate water flows into that ditch.

Farmer John knows not only how many gallons of water each ditch can transport per minute but also the exact layout of the ditches, which feed out of the pond and into each other and stream in a potentially complex network.

Given all this information, determine the maximum rate at which water can be transported out of the pond and into the stream. For any given ditch, water flows in only one direction, but there might be a way that water can flow in a circle.

Digit DP

Posted on 2018-09-14 Edited on 2019-04-17 In Dynamic Programming

Description

Digit DP is a type of dynamic programming often associated with stunningly high data ranges (by that, I mean $10^{100000}$). It is often used for counting the number of integers satisfying specific constraints in the range $l$ to $r$. As implied by its name, digit DP is based on digits: units, tens, hundreds, etc. Digit DP, in its essence, is brute-force enumeration with memorisation.

Cartesian Tree

Posted on 2018-09-13 Edited on 2019-04-17 In Data Structures

Description

This post is closely related to the post on RMQ. Cartesian Tree is a type of binary tree developed by Vuillemin (1980). It is built from an array, and allows searching for the maximum (minimum) element in range as well as lowest common ancestor. The three properties below uniquely define a Cartesian tree:

  1. Node: each element in the array corresponds to exactly one node in the Cartesian tree, similarly, each node in the Cartesian tree corresponds to exactly one element in the array.
  2. In-order traversing: the in-order traverse of the Cartesian tree gives the original array. This implies that for any node, the index (in the array) of its left child is smaller than its index, and that the index of its right child is higher than its index.
  3. Treap: the value corresponded by any node is higher (lower) than all values in its left and right sub-trees.

Game Theory in Competitive Programming

Posted on 2018-09-09 Edited on 2019-04-17 In Number Theory and Mathematics

Description

Game theory investigates the interaction of rational beings in different scenarios. Its application is most notable in economics and other social sciences, although it is extended into computer science and logic.

The typical game theory problems in competitive programming often require the program the determine the guaranteed winner of the game given certain information. Game theory problems do not appear frequently in programming contests, and when they do, they are often the Ad Hoc problem. Nonetheless, I personally find Game Theory, especially the Sprague-Grundy theorem, to be interesting. This post first introduces the reader to relatively simple games to help build the basic intuition. It then explains the Sprague-Grundy theorem with examples. In the end, several exercise problems are given, with solution or hint.

Introduction to Functions

Functions in computer science are not the same as functions in Maths. Think about functions as sub-programs within a program. Functions allow us to organise code doing different things into different functions. We can also use functions multiple times for clarity: originally if we want to use the same code three times in different parts of the program, we would need to copy paste that code, which looks messy; now, we can put that code into a function and call to it three times in the main function.
